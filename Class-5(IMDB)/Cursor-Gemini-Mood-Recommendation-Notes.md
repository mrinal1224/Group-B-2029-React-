## Mood-Based Movie Recommendations (Gemini) — Notes (Built with Cursor)

### Feature objective
Build a **Mood Selector** page where a user:
- selects a mood (Chill, Hype, Romance, etc.)
- clicks a button
- gets **5 movie recommendations** generated by Gemini

This version recommends movies **based on mood only** (not limited to the watchlist).

---

### 1) UI component: `MoodSelector.jsx`
File: `src/components/MoodSelector.jsx`

What it does:
- renders a dropdown of moods
- has a “Recommend 5” button
- shows loading/error states
- renders a list of recommendations (title + optional year + reason)

Important UI state:
- `mood`: selected mood
- `loading`: disables button while request runs
- `error`: shows API or parsing issues
- `results`: array of recommendations

We also used an `AbortController` pattern:
- if the user clicks multiple times quickly, older requests are aborted

---

### 2) Gemini integration helper: `src/lib/gemini.js`
Instead of putting API code inside the React component, we created a small helper module:

**Why?**
- keeps UI clean
- easier to debug
- reusable (future: other AI features)

Key parts inside `getMoodRecommendations()`:

#### a) API key + model from `.env`
Vite reads env variables starting with `VITE_`.

Required:
- `VITE_GEMINI_API_KEY`

Optional:
- `VITE_GEMINI_MODEL` (defaults to `gemini-1.5-flash-latest`)

We created `.env.example` to document required keys.

#### b) Endpoint + request format
Gemini API call:
- `POST https://generativelanguage.googleapis.com/v1beta/models/{MODEL}:generateContent?key=...`

Request body:
- `contents`: the user prompt
- `generationConfig`: temperature, output tokens, and structured output config

We used:
- `responseMimeType: "application/json"` to encourage JSON-only output

#### c) Prompt design
We wrote a prompt that:
- clearly states the mood
- forces exactly 5 items
- forces strict JSON output
- keeps reasons spoiler-free

We also asked for variety:
- “Randomize movies”
- “Suggest from different languages”

#### d) Parsing + validation
Real-world issue: models sometimes return text + JSON, or invalid JSON.

To handle that, we improved parsing:
- try `JSON.parse()` first (when response is pure JSON)
- else fallback to `extractJson()` to grab the first `{...}` JSON object from the text

When parsing fails, we throw an error with:
- `finishReason`
- a short response snippet

This is extremely helpful when debugging API output.

---

### 3) Debugging the Gemini errors we hit (and how we fixed them)

#### Error A: 404 model not found
We initially used a model alias that wasn’t available for `generateContent` on the key/account.

Fix:
- Made the model configurable via `.env` as `VITE_GEMINI_MODEL`
- Set a safe default: `gemini-1.5-flash-latest`

This keeps the code stable even if model names change.

#### Error B: “Unexpected response format”
This happened when Gemini didn’t return the exact JSON shape we expected.

Fix:
- More robust JSON parsing (parse → extract JSON)
- Better error messages including a snippet of the raw response
